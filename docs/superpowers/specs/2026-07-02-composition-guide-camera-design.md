# 構図ガイドカメラアプリ 設計書

日付: 2026-07-02
発端: 韓国アプリ「영뽀또」（作例テンプレオーバーレイカメラ、272万表示バズ）に上乗せした構図ガイドカメラを作る。
既存構想: `~/.claude/projects/-Users-yumac/memory/project_composition_guide_app.md`

## コンセプト

カメラ映像に構図理論のガイドをオーバーレイし、**被写体を自動検知して「状況に一番近い構図」を推薦し、合致したら光って知らせる**カメラアプリ。영뽀또が「作家のテンプレを自分でなぞる」受動型なのに対し、こちらはアプリが能動的にフォローする。

## 開発方針

1. **デスクトップ（Webカメラ）で試作**し、UIを煮詰める
2. その後**スマホ対応**（レスポンシブ＋背面カメラ）
3. 将来的にiPhoneネイティブ化を判断

技術: 依存最小のWebアプリ（getUserMedia + Canvas + TensorFlow.js）。GitHub Pages配布可能な構成（X-Image Frame / Film Scanner と同路線）。

## 機能

### 1. 画面構成
- 全画面カメラプレビュー、その上にCanvasガイドオーバーレイ
- 下部: ガイド選択の横スクロールサムネイル（영뽀또と同じUI文法）
- 上部: アスペクト比切替（4:3 / 16:9 / 1:1）、ガイド線の濃度調整
- シャッターボタン

### 2. 構図ガイド（2階層）
- **基本4種**: 三分割 / 三角構図（対角線＋直交補助線） / 黄金比（分割線＋スパイラル） / 白銀比（1:1.414）
- **応用5種**: ファイグリッド / レイルマン構図 / 対角線 / シンメトリー / 日の丸
- 各ガイドは反転・回転対応（スパイラル4方向など）
- ガイドは「線・円・曲線＋スイートスポット座標」の**データ定義**として分離。構図追加＝定義追加のみ。ネイティブ移植時もデータ流用可

### 3. 自動被写体検知＋合致サイン
- TensorFlow.js（COCO-SSD or MediaPipe）でリアルタイム被写体検出（人物・食べ物・コップ等）
- 被写体の中心/輪郭がガイドのスイートスポット（交点・円）に入ると、円がふわっと光る（スマホ時はハプティクス）
- 負荷対策: 検出は数フレームおき、精度・負荷はデスクトップ試作で検証
- フォールバック: 検出困難なシーンは手動タップで被写体指定

### 4. おまかせ構図レコメンド（영뽀또への最大の上乗せ）
- 検出被写体の数・位置・大きさを各ガイドのスイートスポット配置と照合しスコアリング
- 最高スコアの構図を「この構図が近い」と提示し、そのままフォロー開始
- 手動で別ガイドへの切替も常時可能

### 5. 撮影
- シャッターで**ガイドなしの素の映像**をキャプチャ保存
- 練習用オプションとしてガイド焼き込み保存も用意

### 6. 将来枠（今回は受け皿のみ）
- 自作テンプレ: 好きな写真を読み込んで構図抽出→ガイド化
- AI方向指示: 「もう少し右」「寄って」等の能動誘導
- 推奨ズーム誘導（영뽀또の3.1x表示のような）

## アーキテクチャ

```
index.html
src/
  camera.js      … getUserMedia管理（デスクトップ/スマホのカメラ切替）
  guides/        … ガイド定義データ（1構図=1定義オブジェクト）
  renderer.js    … Canvas描画（ガイド線・スイートスポット・合致発光）
  detector.js    … TensorFlow.js被写体検出ラッパー（頻度制御込み）
  recommender.js … 被写体配置×ガイドのスコアリング
  capture.js     … 撮影・保存
  ui.js          … サムネ選択・アスペクト比・濃度
```

各モジュールは独立してテスト可能。guides/のデータとrecommenderのスコアリングはUI非依存で、ネイティブ移植時の資産。

## エラー処理
- カメラ拒否/未接続: 案内表示＋再試行ボタン
- TensorFlow.jsロード失敗/低速端末: 検出なしモード（ガイドのみ）に自動フォールバック
- HTTPS必須（getUserMedia制約）: localhost開発、公開はGitHub Pages

## テスト
- ガイド定義→描画のスナップショット確認（各構図・各アスペクト比）
- recommenderの単体テスト（被写体配置パターン→期待構図）
- 実機確認: デスクトップWebカメラ→iPhone Safari

## 成功基準
- デスクトップで: ガイド9種切替、被写体自動検知、合致発光、レコメンドが動く
- 実際にテーブルフォトを「フォローされながら」撮れる体験が成立する
