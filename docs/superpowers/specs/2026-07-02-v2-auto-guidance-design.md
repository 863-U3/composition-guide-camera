# v2: 知識ゼロ全自動誘導 設計書

日付: 2026-07-02
発端: ゆうさんの本来の理想は「知識がなくても美しい構図で撮れる」。v1は構図を知る人の道具になっていた。線はあくまでアシスト。

## コンセプト転換

**v1**: 人が構図を選ぶ → 丸に合わせる → 光る
**v2**: カメラを向けるだけ → アプリが写ってるものから一番近い美しい構図を自動選択 → 配置・大きさ・傾きの3軸を能動的に誘導 → 光ったら撮る

## 機能

### 1. おまかせがデフォルト・構図自動切替
- 起動時からおまかせON。バッジのタップ不要で**自動で構図が切り替わる**（既存のstableTop安定化＋5s抑制を流用。切替時は「日の丸構図」と1.5秒トースト表示のみ）
- サムネ帯は普段非表示。「構図を選ぶ」ボタンで開閉（プロモード）。手動で選んだらおまかせは自動OFF（トグルで復帰可）

### 2. 誘導矢印（配置）
- 主被写体（最大bbox、手動指定優先）の中心から**最寄りスイートスポットへ矢印**を描画
- 距離に応じて矢印の長さ・透明度が変化、スポット半径内に入ったら矢印消滅→金色発光（既存）
- 純関数 `guidanceVector(subject, variant) -> {dx, dy, dist, spotIndex} | null` をrecommender/hittest系に追加（テスト対象）

### 3. 寄り引き誘導（大きさ）
- 主被写体のbbox面積比で判定: 画面の4%未満→「もっと寄って」、55%超→「少し引いて」、それ以外は無言
- 純関数 `sizeAdvice(subject) -> 'closer' | 'back' | null`（閾値は定数化、テスト対象）
- 表示は画面下部に小さく日本語テキスト（矢印と同時に出る場合は両方）

### 4. 水平誘導（傾き）
- DeviceOrientationで端末ロール角を取得。±2°超で画面中央に水平線インジケータ（傾いた線＋水平の基準線）、±2°以内で緑に変わり1秒後フェード
- iOS 13+は `DeviceOrientationEvent.requestPermission()` が必要＝ユーザー操作起点。初回に「水平ガイドを使う」小ボタンで許可を取る。拒否/非対応なら機能を出さない（静かに無効）
- 角度→表示状態の純関数 `levelState(rollDeg) -> {level: boolean, angle: number}`（テスト対象）

### 5. UI整理
- 線は薄め（デフォルト濃度を下げる）、誘導矢印・発光・水準器を主役に
- 上部バーは折りたたみ（詳細設定: アスペクト比・濃度・焼き込みは「⚙」内へ）。メイン画面は「カメラ・誘導・シャッター」だけ

## 実装方針
- 既存モジュール流用。新規は guidance.js（誘導ベクトル・サイズ判定・水平判定の純関数群）と、renderer拡張（矢印・水準器・トースト描画）
- テスト: 純関数はnode:testでTDD。描画・ジャイロはCDP/実機で目視ゲート
- 制約継続: 依存なし・正規化座標・日本語UI・無料・外部送信なし

## 成功基準
何も知らない人がURLを開いて、**一度も設定を触らず**に「矢印に従って動かす→光る→撮る」が成立する。
